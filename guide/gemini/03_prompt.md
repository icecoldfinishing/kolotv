Analyse globale du projet

Objectif : Générer une description complète du projet pour faciliter sa compréhension et les modifications futures.

Instructions :
1. Décrire l’objectif principal du projet.
2. Donner un aperçu de l’architecture générale (ex: couches, modules, backend, frontend, base de données).
3. Lister les composants principaux et leur rôle (ex: modules, services, classes importantes, tables principales).
5. Documenter les technologies et frameworks utilisés (ex: langage de programmation, serveur d’application, base de données, version).
6. Ajouter les points particuliers ou pièges connus à surveiller (ex: login, URL de connexion, validations, relations entre tables).
7. Fournir des recommandations pour la maintenance ou l’évolution du projet.
8. Format clair et structuré, en texte, pour servir de référence future.

Sauvegarder le résultat dans un fichier texte nommé :@guide/gemini/04_analyse_projet_socobis.md
